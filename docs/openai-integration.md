# OpenAI 연동 아이디어 (보관용)

실제 API 연동 전 **정책·비용·검수 프로세스**를 정한 뒤 단계적으로 도입한다.

---

## 1. 개인화 피드백 (결과 이후)

- **입력**: 과목별 정답률, 오답 문항 id·카테고리, 난이도 분포(로컬 데이터만).
- **출력**: 3~5문장 학습 조언, 다음 세션 추천(예: “네트워크 기초 용어 위주로 1세트”).
- **주의**: 합격/불합격 **판정 문구**는 규칙 기반 유지, LLM은 **조언**만.

---

## 2. 해설 보강 (배치)

- **입력**: 기존 JSON `question`, `answer`, `explanation`.
- **출력**: “흔한 오답 이유” 한 줄, 용어 짧은 정의.
- **프로세스**: 자동 생성 → 사람 검수 → JSON 반영.

---

## 3. 약점 맞춤 미니 세트 (초안)

- **입력**: 사용자 약점 카테고리, 목표 문항 수.
- **출력**: 4지선다 초안 JSON (창작·기출 미복제 원칙 유지).
- **프로세스**: 반드시 **검수 후** 풀에 merge.

---

## 4. 개념 Q&A 챗 (선택)

- 시험 **문항 정답 직접 요청**은 제한, **용어 설명·학습 방향**만.
- 서버 Route Handler에서만 API 키 사용.

---

## 5. 운영 보조

- 신규 문항 초안, 난이도·중복·민감 표현 스크리닝(보조).

---

## 기술 메모

- **키**: 서버 환경 변수 (`OPENAI_API_KEY`), 클라이언트에 노출 금지.
- **모델**: 비용 대비 `gpt-4o-mini` 등부터 검토.
- **로그**: 사용자 원문 프롬프트 저장 시 개인정보 최소화.
