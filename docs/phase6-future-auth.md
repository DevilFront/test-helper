# Phase 6 이후 — 계정·서버 동기화 (예정)

로컬 전용 기능은 `설정 → 데이터`의 **백업 파일**로 기기 간 옮기기가 가능합니다. 아래는 **클라우드 동기화**를 도입할 때의 참고 안내입니다.

## 인증

- [Auth.js](https://authjs.dev/) (Next.js) 또는 동급 OAuth/이메일 로그인
- 세션: JWT 쿠키 또는 서버 세션

## 저장소

- PostgreSQL 등으로 응시 이력·북마크·설정 동기화 스키마 설계
- 민감정보 최소화, 탈퇴 시 삭제 정책 명시

## 동기화 전략

- 로그인 후 로컬 데이터를 서버로 마이그레이션(병합 규칙 정의)
- 오프라인 우선 + 충돌 시 최신 타임스탬프 등

배포·비용·개인정보 처리방침 확정 후 단계적으로 진행하는 것을 권장합니다.
